입력 항목 안내

  • 자산명: 자산을 구분하는 대표 이름
  • 카테고리: 장비 종류(설정 탭에서 목록/토큰 관리)
  • 사용 분류: 주장비/대여장비/프로젝트장비 등 운영 목적
  • 대여 시작일자: 대여장비일 때 실제 대여 시작일
  • 대여 만료일자: 대여장비일 때 반납/계약 종료 예정일
  • 상태: 사용중/대기/폐기필요/폐기완료 상태
  • 사용자: 장비를 실제 사용하는 사람(사용중일 때 필수)
  • 담당자: 자산 관리 책임자(운영/구매/정산 담당)
  • 부서: 자산이 소속된 조직 또는 팀
  • 위치: 자산의 현재 물리적 보관 위치
  • 제조사: 벤더/브랜드 정보
  • 모델명: 정확한 모델 식별 정보
  • 시리얼번호: 제조사 고유 식별 번호(중복 불가)
  • 자산코드: 내부 자산 식별 코드(설정 탭 양식 기준 자동생성 가능)
  • 구매처: 구매한 업체 또는 파트너사
  • 구매일: 실제 구매 일자
  • 보증만료일: AS/보증 종료 예정일
  • 구매금액: 내부 기준 자산 취득가
  • 메모: 점검, 특이사항, 운영 참고 정보
  • 상태 변경 규칙: 대기/폐기계열은 사용자 자동 해제, 사용중은 사용자 지정 필요

자산코드 설정

  • {CAT}: 카테고리별 토큰값 (설정 탭의 카테고리 관리에서 지정)
  • {USG}: 사용 분류 약어 (예: 주장비=PRI, 서버장비=SRV)
  • {YYYY}/{YY}: 연도 4자리/2자리
  • {MM}/{DD}: 월/일 2자리
  • {SEQ}: 일자 기준 4자리 자동 증가 번호
  • {SEQ4}: 지정 자리수 증가 번호 (예: {SEQ3}, {SEQ5})
  • {RAND4}: 4자리 랜덤 숫자

비밀번호 미설정

Bind 비밀번호는 브라우저 로컬 스토리지에 저장되지 않습니다. 스케줄 저장 시 서버에 암호화되어 저장해 재시작 후에도 재사용할 수 있습니다.

조직도 트리(상하 관계) 표시를 위해 아래 3개 속성을 AD에서 같이 가져와야 합니다.

  • 관리자 DN 속성: 기본 manager (상위 사용자의 DN)
  • 사용자 DN 속성: 기본 distinguishedName (현재 사용자 DN)
  • 사용자 GUID 속성: 기본 objectGUID (중복/변경 보조 식별자)

관리자 메일

기존 만료 리포트 메일 형식으로 지정된 관리자 수신자에게 발송합니다.

  • {DATE}: 기준일(YYYY-MM-DD)
  • {NOTIFY_DAYS}: 만료 예정 판단 기준 일수
  • {CHECKED_LICENSES}: 조회한 전체 라이선스 수
  • {EXPIRING_COUNT}: 만료 예정 사용자 할당 건수
  • {EXPIRED_COUNT}: 이미 만료된 사용자 할당 건수
  • {RECIPIENT_COUNT}: 수신 이메일 주소 수
  • {EXPIRING_ITEMS}: 만료 예정 상세 목록 문자열
  • {EXPIRED_ITEMS}: 만료 상세 목록 문자열

최근 발송: 없음

사용자 메일

사용자별 할당 라이선스 만료 정보를 각 사용자 이메일로 개별 발송합니다.

  • {DATE}: 기준일(YYYY-MM-DD)
  • {NOTIFY_DAYS}: 만료 예정 판단 기준 일수
  • {USER_ID}: 사용자 ID
  • {USER_NAME}: 사용자 이름
  • {USER_EMAIL}: 사용자 이메일
  • {USER_EXPIRING_COUNT}: 해당 사용자의 만료 예정 건수
  • {USER_EXPIRED_COUNT}: 해당 사용자의 만료 건수
  • {EXPIRING_ITEMS}: 해당 사용자의 만료 예정 상세 목록
  • {EXPIRED_ITEMS}: 해당 사용자의 만료 상세 목록
  • {USER_ITEMS}: 사용자 기준 통합 목록

최근 발송: 없음
